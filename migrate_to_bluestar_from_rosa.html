Как перейти с Роса Линукс на BlueStar (arch based)

Несколько слов о Синей Звезде - BlueStar это немецкий дистрибутив на базе arch linux, что для пользователя означает возможность установки около 63000 программ, для сравнения Роса Линукс ограничена числом 19000 (по данным repology).

( свежее ядро, плеер и пасьянсы на месте)

Температура и нагрузка на систему в норме (ctrl+Esc, psensor)

Если вы переходите с новой росы на последнюю версию синей звезды, то миграция пройдет в целом без проблем.

Основные моменты

  1. Всегда делайте на Линукс резервную копию перед вмешательством в систему.
    Копируем содержимое хомяка (домашней папки /home/username) со скрытыми папками, для этого
    в источнике и приемнике обязательно стать владельцем sudo chown $USER:$USER -R .
    для переноса всех файлов на другой диск. Удобно выполнить копирование в файловом менеджере mc

    Обратите внимание, что в зависимости от программ, которые вы используете, может понадобится сделать копию каталога /opt или других каталогов, но домашняя папка должна быть скопирована в любом случае.
  1. BlueStar с наименьшими проблемами устанавливается
    в автоматическом режиме, когда стирается выделенный под систему накопитель.

    В файле /etc/pacman.conf ставим SigLevel=Never TrustAll для всех репозиториев.
  1. После установки возвращаем домашнюю папку /home/username на место из копии, настройки
    программ вроде thunderbird, google chrome и т.п. подхватятся автоматически. Для резервного копирования почты в thunderbird можно поставить расширение import export ng
  2. Настройки grub аналогичны (см про росу) Для обновления grub пользуемся другой командой grub-mkconfig -o /boot/grub/grub.cfg
  3. В Синей звезде используется менеджеры пакетов pacman и yay. Добираем неустановленный софт, например,
    yay -Sy google-chrome или sudo pacman -Sy vlc
    для правки настроек удобно использовать gedit или notepadqq
    Совет - поставить приложение r-linux для восстановления удаленных файлов yay -S r-linux
  4. Ставим центр приложений Discover
    yay -Sy discover
    sudo pacman -Syu packagekit-qt5
    чтобы получить больше тем для окружения plasma и программы, распространяемые через flathub и kde.org
    sudo pacman -S flatpak
    flatpak remote-add --if-not-exists flathub https://dl.flathub.org/repo/flathub.flatpakrepo
    flatpak update
    flatpak remote-add --if-not-exists kdeapps --from https://distribute.kde.org/kdeapps.flatpakrepo
    flatpak update
    пример установки поисковика по файлам
    flatpak install fsearch
    flatpak install --user netbeans
    Автоматическое обновление flatpak (необязательно)

    sudo nano /etc/systemd/system/update-system-flatpaks.service
    содержимое файла
    [Unit]
    Description=Update system Flatpaks

    [Service]
    Type=oneshot
    ExecStart=/usr/bin/flatpak --system update -y

    [Install]
    WantedBy=default.target
    Создаем таймер
    sudo nano /etc/systemd/system/update-system-flatpaks.timer
    Настройка таймера
    [Unit]
    Description=Update system Flatpaks daily

    [Timer]
    OnCalendar=daily
    Persistent=true

    [Install]
    WantedBy=timers.target

    Включение таймера

    sudo systemctl daemon-reload
    sudo systemctl enable update-system-flatpaks.timer
    sudo systemctl start update-system-flatpaks.timer

  5. Если проблемы с выводом звука, то ставим компоненты
    sudo pacman -S pulseaudio-alsa
    sudo pacman -S pulseaudio
    sudo pacman -S pulseaudio-jack
    sudo pacman -S pulseaudio-zeroconf
    sudo pacman -S pulseaudio-equalizer
    sudo pacman -S pavucontrol
    права на папку
    chown username:username /home/username/.config/pulse
    затем перезапуск звука
    pulseaudio -k
    pulseaudio -D
  6. Аналогично winetricks ( программа для настройки запуска win приложений) потребует смены прав
    sudo chown username:username -R /home/username/.wine
  7. Автоматическое подключение к wifi, к сожалению, слетает - в сетевом менеджере ставим подключаться автоматически с приоритетом 0, все пользователи могут подключаться - да, на вкладке wifi сохранить ключ переключаем на сохранить в незашифрованном виде, еще можно на всякий случай обновить 2 сетевые службы
    sudo pacman -Syu wpa_supplicant dhcpd. После данных действий wifi обычно поднимается при загрузке автоматически.

  8. При обновлении системы sudo pacman -Syu возможна ситуация, когда из репозиториев система пытается обновить несуществующий к тому моменту пакет. В этом случае удаляем источник проблемы с зависящими от него программами sudo pacman -Rscn [-Rc] имя_пакета
  9. Cиняя звезда идет без графической оболочки pamac над менеджером пакетов pamac, так что ставим её с помощью yay -S pamac-aur
  10. Для либри офис нужно докачать русский язык
    sudo pacman -Sy libreoffice-fresh-ru

  11. Для приложений, поставляемых в виде образов AppImage можно поставить nx Software Center

    либо AppImage Pool


  12. Немаловажное значение имееют темы оформления и другие компоненты дизайна оконной оболочки. Для этого служит pling Store (appImage). Предварительно нужно поставить пакет ocs-url
  13. Приложения snap удобно ставить из Snap Store, для его установки:
    yaourt -Sy snapd
    sudo systemctl start snapd.socket
    sudo systemctl enable snapd.socket
    snap install core
    snap install snap-store
  14. Для настройки стандартного префикса win32 можно использовать установку библиотек с помощью wt-install-all.sh
  15. Переходим на российские зеркала
    в файл /etc/pacman.d/mirrorlist меняем все на

    далее команда sudo pacman -Syyu
