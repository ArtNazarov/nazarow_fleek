О настройке ROSA LINUX

на момент публикации были недоступны репозитории росы,
меняем на зеркала Яндекса





Репозитории со стандартного сервера можно удалить [ cd /etc/yum.repos.d/ ] или переместить в другой каталог, а потом вернуть

по файловой системе, исправляем для корневого, хомяка, эфи и загрузочного параметры

прописываем в /etc/fstab согласно выводу
lsblk -o NAME,SIZE,TYPE,MOUNTPOINT,UUID
( примонтировать ntfs диске в dolphin! )



в Росе нет flatpak, ставим + добавляем репозиторий + центр приложений discover, обращаем внимание на новые темы оформления в discover



sudo yum install flatpak
sudo flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo
sudo yum install discover

для удобства работы с терминалом в /etc/sudoers

username ALL=(ALL) NOPASSWD:ALL

слегка переделываем grub2 (sudo nano /etc/default/grub )
GRUB_GFXPAYLOAD_LINUX=keep
GRUB_DISABLE_RECOVERY=true
GRUB_DISTRIBUTOR="ROSA"
GRUB_GFXMODE=auto
GRUB_DEFAULT=saved
GRUB_SAVEDEFAULT=true
GRUB_TIMEOUT="120"
GRUB_CMDLINE_LINUX_DEFAULT='splash=silent logo.nologo quiet'
GRUB_DISABLE_OS_PROBER=true
#GRUB_THEME="/boot/grub2/themes/rosa/theme.txt"
GRUB_BACKGROUND="/home/username/b2.jpg"
GRUB_CMDLINE_LINUX="rhgb quiet selinux=0"
GRUB_ENABLE_BLSCFG=true
GRUB_PASSWORD_PROTECT_ONLY_EDITING=true
где GRUB_BACKGROUND прописываем путь до обоев
и обновляем update-grub2

для режима гибернации необходимо прописать параметр resume=UUID=метка swap раздела
в параметр GRUB_CMDLINE_LINUX

варианты настройки процессора ( regim-fedora.sh)
#!/bin/sh
cpupower frequency-set -g powersave

sudo cpupower frequency-set -f 1300MHz
консервативный режим
#!/bin/sh

cpupower frequency-set -g conservative

sudo cpupower frequency-set -f 2200MHz

производительный
#!/bin/sh
cpupower frequency-set -g performance

sudo cpupower frequency-set -f 2600MHz
Служба настраивается по заметке
