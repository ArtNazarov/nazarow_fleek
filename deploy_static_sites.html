Развертывание статических сайтов, инструкция

Публикация (развертывание) статического сайта с помощью Render.com

1. Регистрируемся с помощью Github на https://dashboard.render.com/s
2. Нажимаем New -> Static site ( https://dashboard.render.com/select-repo?type=static )
3. Выбираем проект со статикой и нажимаем Connect (подключить).
4. Заполняем имя проекта на Render, будет создан
поддомен ввида projectname.onrender.com, остальные настройки можно не менять
5. Нажимаем Create site

Публикация статического сайта с помощью Firebase 

1. Регистрируемся с помощью Google на https://console.firebase.google.com
2. Нажимаем Build -> Hosting в меню.
3. Устанавливаем firebase-tools
4. В папке с проектом firebase login, firebase init
5. Выбираем стрелками пункт hosting
6. Выбираем проект из пункта 2, в качестве директории указываем текущую (ставим точку)
7. Вводим команду firebase deploy

Публикация статического сайта на Vercel

1. Регистрируемся  https://vercel.com/dashboard
2. Создаем новый проект Add -> new project,
3. И добавляем репозиторий Github Import Git repository
4. В настройках выставляем Framework - Other, Root Directory - ./
5. Нажимаем Deploy

Публикация статического сайта с помощью Cloud Flare Pages 

1. Регистрируемся на https://pages.cloudflare.com/
2. Переходим в раздел Workers and Pages, на вкладке Pages нажимаем Connect to Git
3. Выбираем репозиторий, заполняем настройки аналогичным образом и выполняем deploy
