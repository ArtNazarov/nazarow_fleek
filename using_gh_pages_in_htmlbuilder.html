Использование Github Pages в HtmlBuilder

Создайте репозиторий для статики, например, username.github.io

Клонируйте репозиторий на свой компьютер

Укажите папку репозитория в специальных настройках генератора

Укажите папку с собранным сайтом в специальных настройках

Перейдите на вкладку со сборкой. Нажмите опубликовать на Github

Будет создан очередной коммит, в который будут добавлены рекурсивно файлы и каталоги из папки сборки в каталог локального репозитория

Чтобы перенести изменения (отправить/запушить коммит) из папки на компьютере на удаленный репозиторий на Github перейдите в Терминал (uxterm, konsole и т.п) и внутри папки с локальным репозиторием введите команду git push -u origin main , подтвердите публикацию изменений введя логин и пароль. Также можно открыть Github Desktop и отправить изменения, используя эту программу, при этом Github Desktop может спросить пароль от хранилища учетных данных

Вы можете настроить хостинг для статических сайтов вроде Vercel, указать другой каталог и собирать сайт в папку произвольного репозитория. После отправки данных на удаленный сервер, хостер переопубликует на своем сервере или в своей CDN ваши файлы, перенеся изменения из удаленного репозитория
